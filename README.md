# yulyrics

OBS 브라우저 소스 URL로 송출하는 가사 오버레이 프로그램.

- 상세 설계: [`docs/PLAN.md`](./docs/PLAN.md)
- 원본 요구사항 · 프리셋 참고 이미지: [`docs/reference/`](./docs/reference/README.md)

현재 상태: **M2 — 가사 붙여넣고 스페이스바로 넘기면 OBS에 나온다**

---

## 설치 없이 바로 쓰기 (포터블)

[릴리스](https://github.com/Redater2254/yulyrics/releases)에서 **`yulyrics-beta.exe`** 를 받아
원하는 폴더에 두고 실행하면 끝이다. 설치 과정이 없다.

곡·프리셋·설정은 **`%APPDATA%\yulyrics`** 에 저장된다.
exe 를 어디에 두고 실행하든 그 자리에 폴더가 생기지 않는다.

> USB 등에 넣어 들고 다니려면 **exe 옆에 `yulyrics-data` 폴더를 직접 만들어** 두면 된다.
> 그 폴더가 이미 있을 때만 그쪽에 저장한다 — **앱이 알아서 만들지는 않는다.**
> (바탕화면에서 실행했다고 바탕화면에 폴더가 생기면 안 되고,
> 그곳이 OneDrive 동기화 폴더라면 설정이 클라우드로 올라가 버린다)

> 코드 서명을 하지 않았으므로 Windows SmartScreen 경고가 뜬다.
> **추가 정보 → 실행**을 눌러야 실행된다.

## 실행 (소스에서)

### 데스크톱 앱 (권장)

```bash
npm install
npm run desktop    # 빌드 후 Electron 실행
npm run dist       # 포터블 exe 빌드 → apps/desktop/release/yulyrics-beta.exe
```

- **창을 닫아도 트레이에 남는다.** 방송 중 창을 정리해도 단축키가 계속 동작한다.
- **전역 단축키** — 다른 창(OBS 등)을 보고 있어도 줄을 넘길 수 있다.

| 기본 단축키 | 동작 |
|---|---|
| `Ctrl + Space` | 앞으로 (다음 줄) |
| `Ctrl + Alt + Space` | 뒤로 (이전 줄) |
| `Ctrl + →` / `Ctrl + ←` | 앞으로 / 뒤로 |

> **수식키(Ctrl·Alt·Shift)가 없는 조합은 걸 수 없다.** 예를 들어 스페이스바 하나를
> 전역 단축키로 잡으면 다른 프로그램에서 스페이스 입력 자체가 막힌다.
> 그래서 뒤로 가기도 `Ctrl+Alt` 만이 아니라 `Ctrl+Alt+Space` 처럼 실제 키가 필요하다.
> (F1~F12 는 예외로 단독 사용이 가능하다)

컨트롤 패널 좌측 「전역 단축키」에서 바꿀 수 있다. 조합을 클릭하고 원하는 키를 누르면 된다.
다른 프로그램이 이미 쓰고 있는 조합은 **「사용 중」으로 빨갛게 표시**된다 —
전역 단축키 등록은 조용히 실패하기 때문에, 방송 중에 눌렀는데 아무 일도 없는 상황을 미리 막는다.

### 서버만 (브라우저에서 조작)

```bash
npm install
npm start          # 빌드 후 서버 기동
```

```
  컨트롤 패널   http://127.0.0.1:7788/
  OBS 오버레이  http://127.0.0.1:7788/overlay
```

## OBS 설정

1. 소스 추가 → **브라우저**
2. URL: `http://127.0.0.1:7788/overlay`
3. 너비 `1920` / 높이 `1080`
4. **「보이지 않을 때 소스 종료」 끄기**, **「장면이 활성화될 때 브라우저 새로고침」 끄기**
5. 사용자 지정 CSS는 **OBS 기본값 그대로** 두기 (건드릴 필요 없음)

크로마키는 필요 없다. 페이지 자체가 완전히 투명하다.
배경이 보인다면 URL에 `?debug=1` 이 붙어 있는지부터 확인할 것.

### OBS가 예전 화면을 보여줄 때

OBS(CEF)는 캐시를 오래 붙들고 있다. 오버레이 번들은 파일명이 `overlay.js` 로 고정이라
해시가 붙지 않으므로, **프로그램을 업데이트해도 OBS만 옛 코드를 계속 돌 수 있다.**
서버가 `cache-control: no-cache` 를 보내도록 해뒀지만, 이미 캐시된 소스는 한 번 비워줘야 한다.

> 브라우저 소스 우클릭 → **속성** → **현재 페이지의 캐시 새로고침**

컨트롤 패널 우상단 배지가 진단을 도와준다.

| 배지 | 뜻 |
|---|---|
| `OBS 1` | 정상 — OBS 브라우저 소스가 붙어 있다 |
| `브라우저 1 · OBS 없음` | 브라우저 탭으로만 열려 있다. OBS에 소스를 추가해야 한다 |
| `오버레이 없음` | 아무것도 연결되지 않았다. URL을 확인할 것 |

우상단 배지의 `오버레이 N` 이 1 이상이면 OBS가 정상 연결된 것이다.

## 쓰는 법

PPT 슬라이드와 같다. **미리 싱크를 찍을 필요가 없다.**

1. 컨트롤 패널 「가사 넣기」에 가사를 한 줄에 하나씩 붙여넣는다
2. **[가사 적용하고 저장]**
3. 방송 중 <kbd>Space</kbd> 로 한 줄씩 넘긴다

가사 입력 문법:

| 쓰기 | 뜻 |
|---|---|
| `전에 자주 비행했었잖아` | 일반 가사 줄 |
| `(간주중)` `(간주)` `(전주)` `(후주)` `--` | 빈 화면 |
| `가사 \|\| 번역` | 번역까지 한 번에 |

노래방 프리셋은 **가사 / 로마자 발음 / 번역** 세 줄을 함께 띄운다.
로마자는 자동 생성되고, 번역이 없는 줄은 그 줄을 아예 그리지 않는다.

빈 화면 마커는 **줄 전체가 괄호로 감싸인 경우에만** 적용된다.
`내 맘속에 (백보컬) 남아` 처럼 가사 안에 있는 괄호는 그대로 나온다.

### 화면 배치 (노래방 프리셋)

두 줄이 좌우로 번갈아 배치되고, 부르는 줄이 흰색으로 강조된다.
빈 자리에는 항상 **다음에 부를 줄**이 미리 대기한다.

```
[ 「친구들과 말썽쟁이」를 부르는 중 ]

   난 그냥 내 삶을 지탱해 줄        ← 연하늘 · 다음 줄 대기
   nan geu nyang ...

              친구들과 말썽쟁이     ← 흰색 · 현재
              chin gu deul gwa ...
```

**단축키**

| 키 | 동작 |
|---|---|
| `Space` `→` `Enter` | 다음 줄 |
| `←` `Backspace` | 이전 줄 |
| `Esc` | 처음으로 (빈 화면) |

> 위 표는 **컨트롤 패널 창 안에서만** 동작하는 키다. `Ctrl` 없이 누르는 것이라,
> OBS 등 다른 창을 보고 있을 때는 먹지 않는다. 그때는 위의 전역 단축키를 쓴다.

**아무리 눌러도 안 넘어간다면** 커서가 이미 곡 끝에 있을 수 있다.
컨트롤 패널 우측 상단의 `3 / 12` 같은 표시를 확인하고, `Esc`(또는 「처음으로」)로 되돌리면 된다.
곡을 새로 불러오면 커서는 항상 시작 전으로 초기화된다.

## 화면 전환 효과

컨트롤 패널 좌측 「화면 전환 효과」에서 바꾼다. **바꾸는 즉시 송출 화면에 반영**되고,
손을 뗀 뒤 프리셋에 저장된다.

| 항목 | 설명 |
|---|---|
| 등장 / 퇴장 | 없음 · 페이드 · 아래에서 위로 · 위에서 아래로 · 좌우 슬라이드 · 작게→제자리 · 통 튀어나오기 · 흐림→또렷 · 눕혔다 세우기 |
| 속도 | 0 ~ 900ms. 퇴장은 등장의 80%로 자동 계산된다 |
| 단어별 | 0이면 줄 전체가 한 번에. 값을 주면 단어가 순서대로 흘러나온다 |

기본 프리셋을 수정하면 **같은 ID로 사용자 사본이 저장**되어 원본을 덮어쓴 것처럼 동작한다.
`[기본값]` 을 누르면 사본이 삭제되고 원래 프리셋이 되살아난다.

> 브라우저에서 조작할 때는 단축키가 **컨트롤 패널에 포커스가 있을 때만** 동작한다.
> 데스크톱 앱(`npm run desktop`)을 쓰면 전역 단축키가 걸려 이 제약이 사라진다.
>
> 스트림덱·AutoHotkey 등에서는 아래 URL을 호출하면 된다 (GET/POST 모두 받는다).
>
> ```
> GET http://127.0.0.1:7788/api/cursor/next
> GET http://127.0.0.1:7788/api/cursor/prev
> GET http://127.0.0.1:7788/api/cursor/reset
> GET http://127.0.0.1:7788/api/state/hidden/toggle
> ```

### URL 파라미터

| 파라미터 | 설명 | 예 |
|---|---|---|
| `preset` | 이 소스에만 고정할 프리셋 ID | `?preset=translation-sub` |
| `layer` | 소스 역할 (`lyrics` / `nowplaying` / `progress`) | `?layer=lyrics` |
| `offset` | 지연 보정 ms. 양수 = 가사가 더 빨리 | `?offset=-120` |
| `token` | LAN 모드에서 요구되는 토큰 | `?token=AB12CD34` |
| `debug` | ⚠️ **개발용.** 배경에 체커보드를 그린다 | `?debug=1` |

> **`?debug=1` 은 OBS URL에 붙이면 안 된다.** 투명 배경을 브라우저에서 눈으로 확인하려고
> 회색 격자를 *일부러* 그리는 옵션이다. 다만 OBS 안(`window.obsstudio` 감지)에서는
> 이 값이 붙어 있어도 무시되므로, 실수로 붙여도 방송에는 나가지 않는다.

장면마다 다른 프리셋을 쓰려면 `?preset=` 을 붙인 URL을 각각 브라우저 소스로 추가하면 된다.

## 개발

```bash
npm run server           # 서버 (tsx watch)
npm run watch:overlay    # 오버레이 자동 재빌드   (별도 터미널)
npm run watch:control    # 컨트롤 패널 자동 재빌드 (별도 터미널)
npm run typecheck        # 전체 타입 검사
```

서버가 빌드 결과물을 직접 서빙하므로 개발 중에도 OBS가 보는 URL과 포트가 동일하다.
(Vite 개발 서버를 따로 띄우지 않는 이유 — OBS에 넣는 주소가 두 개가 되면 혼란스럽다)

## 구조

```
packages/
  core/       타입 계약 · 프리셋 정규화 · LRC · 로마자 변환 · 재생 시계
  presets/    기본 프리셋 2종 + 번들 폰트 (사용자 프리셋과 완전히 같은 포맷)
  renderer/   프리셋 → DOM 렌더러 (오버레이와 프리셋 에디터가 공유)
  server/     Fastify + WebSocket 상태 허브 + 파일 저장소
apps/
  overlay/    OBS 브라우저 소스가 로드하는 페이지
  control/    컨트롤 패널
  desktop/    Electron 셸 (트레이 상주 + 전역 단축키)
```

데스크톱 셸은 **서버를 새로 만들지 않고 `startServer()` 를 그대로 부른다.**
앱과 브라우저의 서버가 갈라지면 "브라우저에서는 되는데 앱에서는 안 되는" 부류의 문제가 생긴다.
단축키도 마찬가지로 `/api/cursor/next` 를 HTTP 로 호출한다 — 컨트롤 패널·스트림덱·단축키가
모두 같은 경로를 지나가야 한다.

사용자 데이터는 개발 중 `userdata/`, 패키징 후 `%APPDATA%/yulyrics` 에 저장된다.

## 폰트

기본 프리셋은 번들 폰트를 쓴다 — 프리셋 1은 **Jua(배민 주아)**, 프리셋 2는 **Montserrat Italic +
Pretendard Black**. 서버가 `/fonts/fonts.css` 로 `@font-face` 를 자동 생성하므로
OBS에서도 별도 설치 없이 그대로 나온다.

번들 폰트는 **전부 SIL OFL 1.1** 이다. 추가할 때의 규칙과 주의사항(특히 웨이트)은
[`packages/presets/fonts/README.md`](./packages/presets/fonts/README.md) 참고.

## 데스크톱 앱 UI

기본 윈도우 타이틀바를 쓰지 않고 **앱 헤더가 곧 타이틀바**다.
헤더 빈 곳을 잡아 창을 끌고, 더블클릭하면 최대화된다. 스크롤바도 앱 톤에 맞춰 바꿨다.

창을 닫아도 **트레이에 남아 전역 단축키가 계속 동작한다.**
완전히 끄려면 트레이 아이콘 우클릭 → 종료. (좌측 「닫아도 트레이에 남기기」로 끌 수 있다)
