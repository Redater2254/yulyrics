# yulyrics — 가사 오버레이 프로그램 개발 계획서

> 작성일: 2026-07-22
> 원본 요구사항 및 프리셋 참고 이미지: [`docs/reference/`](./reference/README.md)

## 1. 프로젝트 개요

### 1.1 한 줄 정의
방송인이 재생 중인 노래의 가사를 **OBS 브라우저 소스 URL**로 실시간 송출하는 데스크톱 프로그램.

### 1.2 핵심 요구사항 (원본 브리프)
1. 가사 오버레이 프로그램
2. **프리셋 시스템** — 예시 프리셋 2종 기본 제공 + **사용자가 직접 제작 가능**
3. **(최우선) OBS 연동** — 로컬 주소(URL) 기반 오버레이

### 1.3 참고 제품: SUBSTAR
| 기능 | SUBSTAR 방식 | yulyrics 방식 |
|---|---|---|
| 송출 | 투명 창(윈도우 캡처) | **로컬 HTTP URL → OBS 브라우저 소스** ✅ |
| 음원 | YouTube URL / 로컬 파일 | 동일 |
| 싱크 | 스페이스바 탭 | 동일 + LRC 임포트/워드 싱크 |
| 커스터마이징 | Pro 전용, 테마 저장 | **전 기능 무료, 프리셋 JSON 공유 가능** |
| 부가 | 키 조절, 곡정보 팝업, 발음 표기 | 동일 (단계적 구현) |

> **차별점**: 투명 창 캡처는 창 포커스/DPI/스케일링 문제가 잦다. URL 방식은 OBS가 직접 렌더링하므로 안정적이고, **장면(Scene)마다 다른 프리셋**을 띄우거나 **다른 PC의 OBS**에서도 받을 수 있다.

> 출처: `docs/reference/preset-2-translation.png`

```
│                                               │
│   I just need someone in my life to give...   │  ← 노랑 볼드 이탤릭
```

## 3. 기술 스택

| 영역 | 선택 | 근거 |
|---|---|---|
| 런타임 | **Node.js 20 LTS** | 로컬 서버 · 파일 접근 · yt-dlp 연동 한 번에 |
| 서버 | **Fastify + `ws`** | 정적 서빙 + REST + WebSocket, 경량 |
| 오버레이 UI | **Vanilla TS + CSS** (프레임워크 없음) | OBS 내부 CEF에서 돌아가므로 CPU 최소화가 최우선 |
| 컨트롤 UI | **Vite + TypeScript** (M1) → **React** (M3~) | M1 컨트롤은 버튼 몇 개뿐. 에디터가 붙는 M3에 React 도입 |
| 패키지 관리 | **npm workspaces** | pnpm 미설치 환경에서도 `npm install` 한 번으로 끝 |
| 데스크톱 셸 | **Electron** (v1) | Node 서버를 그대로 품을 수 있음. Tauri는 sidecar 복잡도 ↑ |
| 스타일 | CSS Modules + CSS 변수 | 프리셋 값을 CSS 변수로 주입 → 리렌더 없이 실시간 반영 |
| 상태 동기화 | WebSocket (JSON) | 컨트롤 ↔ 서버 ↔ 오버레이 N개 |
| 저장 | 로컬 JSON 파일 (`%APPDATA%/yulyrics/`) | DB 불필요, 사용자가 직접 공유·백업 가능 |

> **왜 Electron인가**: 오버레이는 어차피 브라우저에서 도니 데스크톱 셸은 "컨트롤 패널 + 트레이 + 서버 수명 관리"만 하면 된다. 초기에는 셸 없이 `node server.js` + 브라우저로도 100% 동작하도록 만들고, Electron은 마지막에 껍데기로 씌운다. → **셸 종속성 0**

```
   │  컨트롤 패널      │        │  OBS 브라우저 소스      │
   │  · 가사 편집      │        │  · 투명 배경           │
   │  · 탭 싱크        │        │  · 프리셋 렌더링        │
   │  · 프리셋 에디터  │        │  · 카라오케 와이프      │
   │  · 오디오 재생    │        │  (여러 개 동시 가능)     │
   └──────────────────┘        └───────────────────────┘
```

### 4.1 시간 동기화 (드리프트 보정)
와이프 애니메이션이 튀지 않으려면 오버레이가 **자체 시계**를 돌려야 한다.

```
서버 → 오버레이 (200ms마다):  { mediaTimeMs, serverTs, playing, rate }
오버레이: 로컬 rAF 루프로 mediaTime 추정
          오차 > 80ms → 즉시 스냅 / 오차 ≤ 80ms → 3% 씩 완만 보정
```
WS 메시지는 상태 변화 시에만, 시간은 200ms 하트비트로만 → 초당 60프레임을 네트워크로 보내지 않는다.

## 5. ★ OBS 연동 상세 (핵심)

### 5.1 사용자 플로우
1. yulyrics 실행 → 컨트롤 패널 우상단에 URL 표시 + **[복사]** 버튼
2. OBS → 소스 추가 → **브라우저**
3. URL 붙여넣기, 너비 `1920` / 높이 `1080`
4. 끝. 크로마키 불필요 (브라우저 소스는 기본 알파 지원)

### 5.2 URL 스킴
```
http://127.0.0.1:7788/overlay?preset=karaoke-classic&layer=lyrics&token=AB12CD34
```

| 파라미터 | 설명 | 기본값 |
|---|---|---|
| `preset` | 사용할 프리셋 ID. 생략 시 컨트롤 패널에서 선택한 것 | (현재 선택) |
| `layer` | `lyrics` / `nowplaying` / `progress` — 소스별 역할 분리 | `lyrics` |
| `token` | 최초 실행 시 생성되는 로컬 토큰 (LAN 노출 시 보호용) | 자동 |
| `scale` | 캔버스 스케일 배율 (1080p 아닌 캔버스 대응) | `1` |
| `offset` | 지연 보정 ms (예: `-120`) | `0` |

> **layer 분리의 이점**: "가사"와 "곡 정보 팝업"을 OBS에서 각각 다른 위치·다른 장면에 배치할 수 있다.

### 5.3 OBS 브라우저 소스 권장 설정
| 항목 | 값 | 이유 |
|---|---|---|
| 너비 × 높이 | 1920 × 1080 | 프리셋 캔버스 기준. 이후 OBS에서 축소 |
| FPS 제어 | OBS 기준 (보통 60) | 와이프 부드러움 |
| 사용자 지정 CSS | **비우기** | 기본값 `background: transparent` 유지 |
| 보이지 않을 때 소스 종료 | **끄기** | 장면 전환 시 재연결·깜빡임 방지 |
| 장면 활성 시 브라우저 새로고침 | **끄기** | 동일 |
| 페이지 권한 | 기본 | 오디오 미사용 시 불필요 |

### 5.4 다른 PC의 OBS에서 받기 (LAN 모드)
설정에서 `LAN 허용`을 켜면 `0.0.0.0` 바인딩 + LAN IP 안내:
```
http://192.168.0.15:7788/overlay?token=AB12CD34
```
- 토큰 없으면 401 → 우연한 노출 방지
- Windows 방화벽 인바운드 규칙 자동 등록 안내 문구 표시

### 5.5 obs-websocket 연동 (선택, 후순위)
obs-websocket v5로 붙으면 가능해지는 것:
- 곡 시작 시 **장면 자동 전환**
- 가사 소스 **자동 표시/숨김**
- OBS 녹화 시작 시 자동 싱크 리셋

### 5.6 대안 송출 경로 (백업)
URL 방식이 안 되는 환경을 위해 **투명 창 모드**도 옵션으로 제공 (Electron `transparent: true` + `setIgnoreMouseEvents`). 단, v1 범위 밖.

## 6. 데이터 모델

### 6.1 곡 프로젝트 `*.ylrc` (JSON)
```jsonc
{
  "version": 1,
  "meta": { "title": "제목", "artist": "가수", "album": "", "coverPath": "", "durationMs": 213000 },
  "audio": { "type": "local" /* | "youtube" */, "src": "C:/music/a.mp3", "offsetMs": 0 },
  "presetId": "karaoke-classic",
  "lines": [
    {
      "id": "l1",
      "startMs": 12400,
      "endMs": 15800,
      "text": "전에 자주 비행했었잖아",
      "sub": { "romaja": "jeo ne  ja ju  bi haeng hae seot jan a", "translation": "" },
      "words": [                      // 카라오케 와이프용 (선택)
        { "text": "전에",   "startMs": 12400, "endMs": 13100 },
        { "text": "자주",   "startMs": 13100, "endMs": 13900 },
        { "text": "비행했었잖아", "startMs": 13900, "endMs": 15800 }
      ]
    }
  ]
}
```

**임포트/익스포트**
- `.lrc` (표준) 임포트/익스포트
- `.lrc` A2 확장 태그 `<mm:ss.xx>` 단어 싱크 지원
- `.srt` 임포트 (번역 자막용)
- 순수 텍스트 붙여넣기 → 줄 분리 후 탭 싱크

### 6.2 프리셋 `*.ypreset` (JSON)
```jsonc
{
  "id": "karaoke-classic",
  "name": "노래방 클래식",
  "author": "yulyrics",
  "version": 1,
  "canvas": { "w": 1920, "h": 1080 },

  "layout": {
    "mode": "dual-alternate",        // dual-alternate | stacked-center | single-line | scroll
    "anchor": "bottom",              // top | center | bottom
    "offset": { "x": 0, "y": -80 },
    "maxWidthPct": 88,
    "lineGap": 12,
    "blockGap": 28,
    "linesVisible": 2                // 동시에 보이는 가사 줄 수
  },

  "layers": [
    {
      "role": "main",
      "source": "text",
      "font": { "family": "BM JUA", "sizePx": 84, "weight": 700, "italic": false, "letterSpacing": 0 },
      "fill": "#B8DFFF",
      "stroke": { "color": "#1B4FD8", "widthPx": 8, "join": "round" },
      "shadow": [{ "x": 0, "y": 4, "blur": 8, "color": "#00000066" }],
      "opacity": 1
    },
    {
      "role": "sub",
      "source": "romaja",            // romaja | translation | none
      "font": { "family": "Arial", "sizePx": 46, "weight": 400 },
      "fill": "#F2F0C4",
      "stroke": { "color": "#4A3A10", "widthPx": 3 }
    }
  ],

  "karaoke": {
    "enabled": true,
    "mode": "wipe",                  // wipe | jump | glow | none
    "unit": "word",                  // word | syllable
    "sungFill": "#FFFFFF",
    "sungStroke": "#1B4FD8",
    "easing": "linear",
    "leadMs": 0
  },

  "animation": {
    "in":  { "type": "fadeUp", "durMs": 250, "easing": "cubic-bezier(.2,.8,.2,1)" },
    "out": { "type": "fadeOut", "durMs": 200 },
    "idle": "none"
  },

  "background": { "type": "none" },  // none | solid | bar | blur
  "safeArea": { "top": 60, "right": 80, "bottom": 60, "left": 80 }
}
```

**설계 원칙**
- 프리셋은 **순수 데이터**. 코드/스크립트 없음 → 공유해도 안전
- 알 수 없는 필드는 무시, 누락 필드는 기본값 → **버전 호환**
- 기본 프리셋 2종도 이 포맷을 그대로 사용 (특별 취급 없음)

## 7. 프리셋 에디터 (사용자 제작)

### 7.1 UI 구성
```
┌────────────────┬──────────────────────────────┐
│  속성 패널      │   실시간 미리보기              │
│                │   (실제 오버레이와 동일 렌더러)  │
│  ▸ 레이아웃     │                              │
│  ▸ 메인 텍스트   │   [샘플 가사로 애니메이션 반복]  │
│  ▸ 보조 텍스트   │                              │
│  ▸ 카라오케     │   배경: 체커보드 / 단색 / 이미지 │
│  ▸ 애니메이션   │                              │
│  ▸ 배경        │   ─────────────────────────  │
│                │   ◀ 재생바로 시점 스크럽 ▶     │
└────────────────┴──────────────────────────────┘
```

### 7.2 기능
- **복제로 시작**: 기본 프리셋 우클릭 → "복제해서 편집"
- **실시간 반영**: 값 변경 → CSS 변수만 갱신 → 미리보기 + 송출 중인 OBS까지 즉시 반영
- **폰트**: 번들 폰트 목록 + 시스템 폰트 스캔 + `.ttf/.otf` 직접 추가
- **색상**: 알파 포함 컬러 피커, 스포이드
- **내보내기/불러오기**: `.ypreset` 파일 드래그&드롭
- **되돌리기**: Ctrl+Z 히스토리 스택

### 7.3 렌더링 구현 노트
| 항목 | 방식 |
|---|---|
| 두꺼운 외곽선 | **HTML 텍스트 + `-webkit-text-stroke` + `paint-order: stroke fill`** |
| 카라오케 와이프 | 텍스트를 2겹 스택, 상단 레이어에 `clip-path: inset(0 Npx 0 0)` (단어 위치 DOM 실측) |
| 한 줄 맞춤 | `autoFit` — 폭 초과 시 줄바꿈 대신 `transform: scale()` 로 축소 |
| 성능 | `transform`/`opacity`/`clip-path`만 애니메이션, 변화 0.5px 미만이면 DOM 미접촉 |
| 폰트 로딩 | `document.fonts.ready` 이후 실측 → 폭 계산 오류 방지 |

> **결정 (M1에서 수정됨)**: 원래 SVG `<text>` 를 택했으나, **HTML + `paint-order`** 로 변경했다.
> `paint-order: stroke fill` 을 주면 획이 글자 뒤에 그려져 한글이 뭉개지지 않고,
> SVG와 달리 flexbox·줄바꿈·폭 실측을 그대로 쓸 수 있다. OBS는 Chromium(CEF)이라 지원이 보장된다.
> 단, `-webkit-text-stroke` 는 획을 글자 경계 **중앙**에 그리므로 바깥으로 W px 를 보이게 하려면 두께를 **2W** 로 줘야 한다.
> 렌더러가 이 변환을 대신 처리한다 (`cssStrokeWidth`).

## 8. 진행 방식

곡마다 미리 싱크를 찍는 건 준비 부담이 크다. 그래서 **수동 진행을 기본**으로 하고,
자동 타임라인은 이미 싱크 데이터가 있을 때만 쓰는 선택지로 둔다.

### 8.1 수동 모드 (`manual`) — 기본

PPT 슬라이드처럼 사람이 스페이스바로 한 줄씩 넘긴다. **타이밍 데이터가 아예 필요 없다.**

```
1. 가사 붙여넣기    한 줄에 한 소절
                   `--`  만 있는 줄 = 빈 화면 (간주)
                   `가사 || 번역`   = 번역 동시 입력
        ↓
2. [가사 적용]      즉시 저장되고 오버레이에 연결됨
        ↓
3. 방송 중          [Space] 로 한 줄씩 넘김. 끝.
```

처음 부르는 곡도 가사만 복사해서 바로 쓸 수 있는 게 핵심이다.

**단축키**

| 키 | 동작 |
|---|---|
| `Space` `→` `PageDown` `Enter` | 다음 줄 |
| `←` `PageUp` `Backspace` | 이전 줄 |
| `Esc` | 처음으로 (빈 화면) |
| 줄 목록 클릭 | 그 줄로 점프 |

**외부 도구 연동** — 스트림덱·AutoHotkey 등은 URL 하나만 호출할 수 있는 경우가 많아
`GET` 도 함께 받는다.

```
GET /api/cursor/next    다음 줄
GET /api/cursor/prev    이전 줄
GET /api/cursor/reset   처음으로
```

**노래방 프리셋에서의 표현** — 줄 안쪽 진행률이라는 개념이 없으므로,
현재 줄은 통째로 '부른 색(흰색)', 다음 줄은 '안 부른 색(연하늘)'로 그린다.
어느 줄을 부르는 중인지가 한눈에 보인다.

### 8.2 자동 모드 (`timeline`) — 후순위

줄마다 시각이 찍혀 있을 때만 동작한다. 카라오케 와이프(글자 단위 진행)가 가능하다.
`.lrc` 임포트로 데이터가 들어오는 경우가 주 용도이고,
직접 싱크를 찍는 에디터는 **로드맵 맨 뒤**로 미뤘다 (12절 참고).

**로마자 변환**: 국립국어원 로마자 표기법(Revised Romanization) 기반 자체 구현.
음절 단위 분해 후 초성/중성/종성 매핑 + 연음·자음동화 규칙 적용. 결과는 항상 **수동 수정 가능**.

## 14. M1 구현 기록

M1을 만들면서 계획과 달라진 것, 그리고 실제로 부딪힌 버그를 남긴다.

### 계획 대비 변경

| 항목 | 계획 | 실제 | 이유 |
|---|---|---|---|
| 패키지 관리 | pnpm | npm workspaces | 환경에 pnpm 미설치. 설치 단계를 하나 줄임 |
| 외곽선 렌더링 | SVG `<text>` | HTML + `paint-order` | 7.3절 참고. HTML 레이아웃을 그대로 쓸 수 있음 |
| 컨트롤 UI | React | Vanilla TS (M3에 React) | M1 컨트롤은 버튼 몇 개. 지금 React를 넣을 이유가 없음 |
| 프런트 서빙 | Vite 개발 서버 | 서버가 빌드 결과물 직접 서빙 | OBS에 넣는 URL과 포트를 개발/운영 동일하게 유지 |

### 실제로 잡은 버그

1. **오버레이가 완전 백지** — Vite `base: './'` 때문에 `/overlay`(슬래시 없음)에서
   `./overlay.js` 가 `/overlay.js` 로 풀려 404. → `base: '/overlay/'` 절대 경로로 고정.
   OBS에 넣는 URL을 짧게 유지하려면 base는 마운트 경로와 같아야 한다.

2. **2행이 우측 정렬되지 않음** — `.yl-block { width: 100% }` 이 슬롯의 `align-items` 를 무력화.
   → 정렬을 `--yl-align` CSS 변수로 블록까지 전달.

3. **지나간 가사가 화면에 남음** — 퇴장 애니메이션의 `finish` 이벤트에만 노드 제거를 걸어뒀는데,
   백그라운드 탭/숨겨진 소스에서는 WAAPI가 멈춰 이벤트가 영영 오지 않음.
   → `setTimeout(durMs + 100)` 가드 추가. **노드 수명을 애니메이션 이벤트에만 맡기면 안 된다.**

4. **rAF 정지 시 화면이 굳음** — 브라우저가 백그라운드 탭의 `requestAnimationFrame` 을 멈춤.
   → 서버 틱(200ms) 수신 시에도 렌더하도록 이중화. OBS에서는 문제가 없지만
   일반 브라우저로 미리 볼 때 "고장난 것처럼" 보이는 걸 막는다.

### 알려진 한계 (M1 시점)

- **로마자 발음**: 연음만 처리한다. 비음화(`끝나면`→`끈나면`), 구개음화(`맏이`→`마지`) 등은
  형태소 분석이 필요해 미구현. 결과는 항상 수동 편집 가능하다.
- **음원 없음**: 재생은 데모(서버 타이머)와 수동 스크럽만 가능. 실제 오디오는 M4.

## 14.5 폰트 번들 (M1 후속)

### 폰트 식별

참고 이미지의 프리셋 1 폰트를 후보 6종과 실제 렌더링해 대조한 결과 **Jua(배달의민족 주아)** 로 판정.
ㅈ의 굽은 윗획, ㅇ의 원형 카운터, 획 끝 처리, 자소 비율이 모두 일치한다.

프리셋 2는 원어(라틴) **Montserrat Italic**, 번역(한글) **Pretendard Black** 으로 맞췄다.

### 여기서 나온 함정

**한글 디스플레이 폰트는 웨이트가 400 하나뿐이다.** 프리셋에 `weight: 700` 을 준 상태로
Jua 를 적용하니 브라우저가 가짜 볼드를 합성해 원본보다 훨씬 굵게 나왔다.
→ `weight: 400` 으로 수정. 굵기가 필요하면 웨이트를 올리는 게 아니라 굵은 폰트를 고르는 것이 맞다.

**`document.fonts.ready` 만 믿으면 안 된다.** `@font-face` 는 실제로 쓰이는 순간 로드되므로,
웹폰트가 요청되기도 전에 `ready` 가 resolve 될 수 있다. 그 시점에 폭을 재면 대체 폰트 기준
좌표가 잡혀 **카라오케 와이프 경계가 어긋난다.** → `fonts.addEventListener('loadingdone')` 으로 재측정.

`font-display: block` 을 쓴 이유도 같다. 방송 화면에서 대체 폰트로 한 번 그렸다가
바뀌는 것(FOUT)이 잠깐 안 보이는 것보다 훨씬 눈에 띈다.

### 크기 보정

원본에서 가사가 프레임 폭의 약 50% 를 차지하는데 초기값(84px)은 40% 였다.
→ 메인 100px / 보조 54px / 외곽선 9px 로 조정해 47% 로 맞췄다.
크롭 이미지에서 역산한 값이라 정확한 원본 수치는 아니며, 프리셋에서 바로 바꿀 수 있다.

### 라이선스 방침

**OFL 등 재배포가 명시된 폰트만 번들한다.** 로컬 서버가 OBS 로 폰트 파일을 내려보내는 구조라
이건 명백한 재배포다. "무료 사용"과 "재배포 가능"은 다르다.
라이선스 원문은 `packages/presets/fonts/licenses/` 에 폰트별로 동봉했다 (OFL 의무 조항).
사용자가 상용 폰트를 쓰고 싶으면 프리셋 `font.family` 에 시스템 폰트명을 적으면 된다 — 그건 재배포가 아니다.

### 남은 차이

데모 곡이 `text`=한글 / `translation`=영어라, 프리셋 2를 데모로 보면 참고 이미지와 **언어 위치가 반대**다
(원본은 영어가 위, 한글이 아래). 프리셋 자체는 "원어 위 / 번역 아래" 로 올바르고,
어느 언어가 원어인지는 곡 데이터에 달렸다. 다만 한글이 이탤릭 레이어에 오면
합성 이탤릭이 걸려 다소 어색하다 — 그 경우 프리셋에서 italic 을 끄는 것이 낫다.

## 14.6 M2 구현 기록 (수동 진행 모드)

### 설계

`Project.mode` 와 `OverlayState.cursor` 를 추가해, 렌더러가 시간 대신 커서를 따르게 했다.
타임라인 코드를 지우지 않고 분기만 추가했으므로 `.lrc` 를 불러오면 자동 모드도 그대로 동작한다.

`cursor` 는 `-1`(시작 전) ~ `lines.length`(종료) 범위를 갖는다.
양 끝을 "빈 화면"으로 둔 덕분에 곡 시작 전과 끝난 뒤에 가사가 남지 않는다.

### 판단이 필요했던 지점

**`--` 구분자는 화면을 통째로 비운다.** 처음엔 노래방 레이아웃 특성상 다음 줄을
계속 미리 보여줬는데, 그러면 "빈 화면"이라는 이름값을 못 한다.
다음 줄 미리보기가 필요하면 `--` 를 안 넣으면 된다 — 규칙이 단순한 쪽이 예측 가능하다.

**곡을 바꾸면 커서를 항상 -1로 되돌린다.** 이전 곡의 줄 번호가 남아
엉뚱한 가사가 송출되는 것이 최악의 실패다.

**같은 제목이면 같은 곡으로 덮어쓴다.** 오타 하나 고칠 때마다 곡이 새로 쌓이면 목록이 망가진다.

**커서 API는 `GET` 도 받는다.** 스트림덱·AutoHotkey 류는 URL 하나만 호출할 수 있는
경우가 많아서, `POST` 만 열어두면 붙일 방법이 없다.

**버튼에서 포커스를 뗀다.** 버튼을 마우스로 누르면 포커스가 남아,
다음 스페이스바가 그 버튼을 다시 누른다. `mouseup` 에서 `blur()` 로 해제했다.

### M2 이후 실사용에서 잡은 것들

**퇴장 블록이 레이아웃을 밀고 있었다.** 좌우 교대 배치에서 N번 줄이 빠지는 슬롯에
곧바로 N+2번 줄이 들어오는데, 퇴장 애니메이션 중인 블록이 자리를 계속 차지해
두 줄이 잠깐 쌓였다. 실측으로 슬롯 높이가 **36px → 77px** 로 튀었다 —
줄을 넘길 때마다 자막이 밀렸다 돌아오는 원인이었다.
→ 퇴장 블록을 `position: absolute` 로 띄워 레이아웃에서 분리.

**OBS가 옛 코드를 계속 돌고 있었다.** 오버레이 번들은 파일명이 `overlay.js` 로 고정(해시 없음)인데,
OBS(CEF)는 캐시를 오래 붙든다. 그래서 서버는 최신인데 **OBS에서만 수동 모드가 동작하지 않았다.**
"자막 표현이 이상하다"는 증상의 실제 원인이 이것이었다.
→ 프런트엔드 정적 파일에 `cache-control: no-cache`. 이미 캐시된 소스는 사용자가 한 번 비워야 한다.

**연결 배지가 거짓말을 할 수 있었다.** 탭이 강제 종료되면 close 이벤트가 안 와서
소켓이 그대로 남는다. 진단하려고 만든 표시가 거짓말을 하면 없느니만 못하다.
→ 15초 주기 ping/pong 으로 죽은 연결 회수.

**미리보기가 연결 수에 잡히면 안 된다.** 컨트롤 패널에 오버레이를 iframe 으로 끼워 넣었는데,
그게 오버레이로 집계되면 "OBS가 붙었나" 배지가 무의미해진다.
→ `?preview=1` → `role: 'preview'` 로 식별, 카운트에서 제외.

**빈 화면 마커는 사람이 쓰던 대로 받는다.** `--` 만 받으면 규칙을 외워야 한다.
`(간주중)` `(간주)` `(전주)` `(후주)` 도 받되, **줄 전체가 괄호로 감싸인 경우에만** 적용한다.
가사 안의 `(백보컬)` 까지 지워버리면 안 된다.

### 화면 전환 효과

애니메이션 시스템은 M1부터 스키마에 있었지만 **고를 방법이 없었고** 종류도 빈약했다.
효과를 9종으로 늘리고, 컨트롤 패널에서 바로 바꿀 수 있게 했다.

**단어별 지연(stagger)에서 걸린 것 두 가지**

1. `transform` 은 inline 요소에 적용되지 않는다. 단어 span 을 `inline-block` 으로 바꿔야 하는데,
   항상 그렇게 두면 평소 텍스트 렌더링에 괜한 변수가 생긴다.
   → stagger 를 켤 때만 `.yl-stagger` 클래스로 전환한다.
2. 카라오케 와이프는 base 레이어와 sung 레이어가 **픽셀 단위로 겹쳐 있어야** 성립한다.
   base 의 단어만 애니메이션하면 와이프 경계가 어긋난 채로 보인다.
   → 두 레이어의 대응하는 단어를 **같은 지연으로 함께** 움직인다.

**기본 프리셋 수정 방식** — 같은 ID로 사용자 사본을 저장한다.
`listPresets()` 가 사용자 것을 우선하므로 덮어쓴 것처럼 동작하고,
사본을 지우면 코드에 있는 원본이 그대로 되살아난다. "초기화" 기능이 공짜로 생긴다.

편집 중에는 `previewPreset` WS 로 화면만 갱신하고, 손을 뗀 뒤 600ms 지나서 한 번 저장한다.
슬라이더를 드래그하는 내내 파일을 쓰면 안 된다.

## 14.7 M4 구현 기록 (Electron 셸)

### 원칙

**서버를 하나만 유지한다.** 셸은 `startServer()` 를 그대로 부르고, 단축키도
`/api/cursor/next` 를 HTTP 로 호출한다. 앱 전용 경로를 만들면
"단축키로는 되는데 버튼으로는 안 된다" 같은 문제가 반드시 생긴다.

### 리팩터링이 필요했던 것

**서버 진입점이 CLI 였다.** `packages/server/src/index.ts` 가 top-level await 로
스스로 서버를 띄우고 있어서, 셸이 import 하는 순간 포트를 알기도 전에 시작돼 버렸다.
→ `server.ts`(라이브러리) / `cli.ts`(CLI) / `index.ts`(부수효과 없는 재수출) 로 분리.

**경로를 import 시점에 상수로 굳혀두고 있었다.** 패키징하면 리소스가 전혀 다른 위치로
가는데, 모듈 로드 시점에 고정하면 셸이 위치를 알려줄 기회가 없다.
추가로 esbuild 로 CJS 번들하면 `import.meta.url` 이 비어 경로 계산이 통째로 깨진다.
→ `overlayDist()` `dataDir()` 처럼 전부 지연 계산 함수로 바꾸고, `setAppRoot()` / `setDataDir()` 로 주입.

### 판단한 것들

**단축키 등록 실패를 반드시 UI로 되돌린다.** `globalShortcut.register()` 는 조용히 false 를
반환한다 — 다른 프로그램이 그 조합을 쓰고 있으면 눌러도 아무 일이 없다.
**방송 중에 눌렀는데 아무 일도 안 일어나는 것이 가장 나쁜 실패**라,
결과를 `/api/status` 로 흘려보내 컨트롤 패널이 「사용 중」으로 표시하게 했다.

**수식키 없는 단일 키는 전역 단축키로 못 걸게 막았다.** 스페이스바 하나를 전역으로 잡으면
다른 프로그램에서 타이핑이 안 된다. F1~F12 만 예외로 뒀다.
같은 이유로 `Ctrl+Alt` 처럼 수식키만 있는 조합도 등록할 수 없다 — OS 가 단축키로 받지 않는다.
그래서 뒤로 가기는 `Ctrl+Alt+Space` 로 잡았다 (앞으로 `Ctrl+Space` 에 Alt 를 더한 형태).

**기본값이 바뀔 때 사용자가 바꾼 설정은 건드리지 않는다.** 저장된 단축키가 예전 기본값과
정확히 일치할 때만 새 기본값으로 옮긴다(`migrateHotkeys`). 업데이트가 사용자 설정을
멋대로 되돌리면 안 된다.

**개발 중에는 리포지토리 `userdata/` 를 그대로 쓴다.** 설치본만 `app.getPath('userData')` 로 간다.
안 그러면 `npm start` 로 만든 곡이 데스크톱 앱에서 안 보여 혼란스럽다.

**창 닫기 ≠ 종료.** 방송 중 창을 정리했다고 단축키까지 죽으면 안 된다.
트레이에서 「종료」를 눌렀을 때만 실제로 끝난다.

### 남은 한계

**인스톨러를 실제로 만들어보지 않았다.** `electron-builder` 설정(`extraResources` 로
오버레이·컨트롤 빌드와 폰트를 `resources/app` 아래로)은 넣었지만 `npm run dist` 로
패키징해 설치까지 검증하지는 않았다. 배포 전에 반드시 한 번 돌려봐야 한다.

**트레이 아이콘 파일이 없다.** `assets/tray.png` 가 없으면 OS 기본 아이콘이 붙는다.
